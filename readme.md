# Game T14 - GUI, Terminal, and Game Tree Search Bot

Game T14 is a two-player strategy game played on a binary sequence consisting of `0`s and `1`s.  
Players take turns selecting adjacent pairs on the board. Each selected pair is replaced by a single new value, and the selected pattern also affects the players' scores.

This project includes:

- a **main GUI version** implemented with **Tkinter**
- a **terminal version** for direct text-based play
- a **Minimax bot**
- an **Alpha-Beta Pruning bot**
- **game tree visualization** in the terminal version
- utility modules for state copying, move simulation, evaluation, and tree construction

The GUI version is the main playable interface of the project.  
The terminal version is especially useful for testing, debugging, and analyzing the search tree generated by the bot.

---

## 1. Game Overview

The game starts with a randomly generated binary board whose size is between **15 and 25**.

Example board:

```text
0 1 1 0 0 1 0 1
```

On each turn, the active player selects **two adjacent cells**.  
That pair is replaced with **one new value**, so the total board length decreases by 1 after every move.

The game continues until only **one value** remains on the board.

At the end:

- the player with the higher score wins
- if both scores are equal, the result is a draw

---

## 2. Game Rules

The rules depend on:

1. which adjacent pair is selected
2. which player's turn it is

The project implements the scoring from the **active player's perspective**.

### Pair transformation rules

| Pair  | New Value | Score Effect        |
| ----- | --------: | ------------------- |
| `0 0` |       `1` | +1 to active player |
| `0 1` |       `0` | +1 to opponent      |
| `1 0` |       `1` | -1 from opponent    |
| `1 1` |       `0` | +1 to active player |

---

## 3. Detailed Turn Logic

### If it is Player 1's turn

| Selected Pair | New Value | Score Update          |
| ------------- | --------: | --------------------- |
| `0 0`         |       `1` | `player1_points += 1` |
| `0 1`         |       `0` | `player2_points += 1` |
| `1 0`         |       `1` | `player2_points -= 1` |
| `1 1`         |       `0` | `player1_points += 1` |

After the move, the turn changes to **Player 2**.

### If it is Player 2's turn

| Selected Pair | New Value | Score Update          |
| ------------- | --------: | --------------------- |
| `0 0`         |       `1` | `player2_points += 1` |
| `0 1`         |       `0` | `player1_points += 1` |
| `1 0`         |       `1` | `player1_points -= 1` |
| `1 1`         |       `0` | `player2_points += 1` |

After the move, the turn changes to **Player 1**.

- visualizing the game tree

Contains the `GameBot` class.

This class is responsible for:

- selecting moves for the bot

- building the search tree
- collecting search statistics such as:
  - generated nodes
  - evaluated leaf nodes
  - move time

### `core_funcs.py`

Contains the `Game_T14` class.

This is the main game engine and is responsible for:

- creating a new board
- creating a new game
- applying a move to the actual game state

Contains helper functions used mainly by the bot and tree search.

Related to tree printing and tree-visualization support.

## 6. Main Classes and Functions

## `GameObject` (`game_object.py`)

Represents a single snapshot of the game state.

### Fields

- `game_board`  
  Current board as a list of integers

- `board_size`  
  Current board length

- `turn`  
  Current active player
  - `0 = Player 1`
  - `1 = Player 2`

- `player1_points`  
  Score of Player 1

- `player2_points`  
  Score of Player 2

- `turn_count`  
  Total number of moves made so far

### Method

#### `copy()`

Returns a copied `GameObject`.

This is essential for tree search because Minimax and Alpha-Beta simulate many future moves.  
Those simulations must not change the real game currently displayed to the user.

---

## `TreeNode` (`tree_node.py`)

Represents one node in the game tree.

### Fields

- `state` вҖ” the `GameObject` stored at this node
- `move` вҖ” the move index that led to this node
- `parent` вҖ” parent node
- `depth` вҖ” node depth in the tree
- `children` вҖ” list of child nodes
- `value` вҖ” evaluation result assigned by Minimax or Alpha-Beta

### Method

#### `create_child(state, move, parent, depth)`

Creates a new child node, appends it to the current node's `children` list, and returns the created node.

---

## `Game_T14` (`core_funcs.py`)

The main game engine.

### Methods

#### `create_new_board(board_size)`

Generates a random binary board of the requested size.

#### `create_new_game(board_size)`

Creates a new game and initializes `self.game_object`.

#### `turn(move)`

Applies the given move to the real game state.

This function:

- checks move bounds
- reads the selected adjacent pair
- updates scores according to the rules
- replaces the selected pair with a single new value
- decreases board size by 1
- switches the active player
- increments turn count

#### `check_is_end()`

Checks whether the game is over.

Returns:

- `-1` вҶ’ game continues
- `0` вҶ’ Player 1 wins
- `1` вҶ’ Player 2 wins
- `2` вҶ’ draw

#### `print_board()`

Prints the current board to terminal.

#### `print_status()`

Prints:

- Player 1 score
- Player 2 score
- current active player

---

## `GameBot` (`bot.py`)

Controls the decision-making of the bot.

### Constructor Parameters

- `algorithm`
  - `0` вҶ’ Minimax
  - `1` вҶ’ Alpha-Beta

- `max_depth`
  Maximum search depth

- `bot_player`
  Which player the bot controls
  - `0` вҶ’ Player 1
  - `1` вҶ’ Player 2

---

### Main Methods

#### `choose_move(state)`

Selects the best move for the current state.

Depending on the chosen algorithm, it calls either:

- `minimax_decision(...)`
- `alphabeta_decision(...)`

Returns:

```python
(best_move, best_value, stats, root)
```

Where:

- `best_move` вҶ’ chosen move index
- `best_value` вҶ’ evaluation score of the best move
- `stats` вҶ’ dictionary containing:
  - `nodes_generated`
  - `nodes_evaluated`
  - `move_time`
- `root` вҶ’ root node of the generated search tree

---

#### `minimax_decision(state, max_depth, bot_player)`

Starts a Minimax search from the root state.

This method:

- creates the root tree node
- generates the legal root moves
- simulates each move
- recursively evaluates children
- selects the best move
- stores the final root value
- records search time

---

#### `evaluate_node(node, max_depth, bot_player, stats)`

Recursive Minimax evaluator.

Logic:

- if the node is terminal or depth-limited, evaluate the state directly
- if it is the bot's turn, maximize the evaluation
- otherwise, minimize the evaluation

This method also updates:

- node values
- generated node count
- evaluated leaf count

---

#### `alphabeta_decision(state, max_depth, bot_player)`

Starts an Alpha-Beta search from the root.

Similar to `minimax_decision`, but uses:

- `alpha`
- `beta`

to prune unnecessary branches.

---

#### `evaluate_node_ab(node, max_depth, bot_player, stats, alpha, beta)`

Recursive Alpha-Beta evaluator.

Logic is similar to Minimax, but branches are cut when:

```text
beta <= alpha
```

This reduces the number of explored nodes while preserving the optimal result.

## 7. Utility Functions (`game_state_utils.py`)

### `get_legal_moves(state)`

Returns all valid move indices:

```python
[0, 1, 2, ..., len(board)-2]
```

Because each move selects a pair of adjacent cells, the last valid move is always `len(board) - 2`.

---

### `apply_move(state, move)`

Creates a copy of the current state and applies the move to that copy.

This is used during search so that simulated moves do not damage the original game state.

It performs the same rule logic as the real game engine:

- reads selected pair
- updates scores
- replaces the pair
- switches turn
- increments turn count

Returns the new copied state.

---

### `evaluate_state(state, bot_player)`

Returns the heuristic evaluation of a state from the bot's perspective.

If the bot is Player 1:

```python
state.player1_points - state.player2_points
```

If the bot is Player 2:

```python
state.player2_points - state.player1_points
```

So the bot always tries to maximize:

```text
bot_score - opponent_score
```

## 8. Terminal Tree Visualization

One of the most useful features of the terminal version is the printed game tree.

After the bot makes a move, the program can print the first few levels of the search tree.

This is very helpful for:

- understanding Minimax
- understanding Alpha-Beta behavior
- seeing why the bot selected a move
- comparing selected and rejected branches

### Tree markers

- **Green `вҳ…`** вҶ’ applied move branch
- **Blue `в—Ҹ`** вҶ’ same-value alternative branch
- **Red `в—Ӣ`** вҶ’ rejected branch
- **Yellow root** вҶ’ root node

### Information shown per node

Each printed node can include:

- node depth
- move index
- chosen pair
- board contents
- Player 1 score
- Player 2 score
- whether the node is `MAX` or `MIN`
- node evaluation value
- leaf marker

This makes the terminal version extremely useful as an educational search-tree visualizer.
